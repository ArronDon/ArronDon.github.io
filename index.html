Creating Objects through Reflection in Java

转自Creating Objects through Reflection in Java with Examples

Class.newInstance()和Constructor.newInstance()的不同

  • Class.newInstance()只能调用无参构造器，Constructor.newInstance()可以调用任何类型的构造器，无论参数有几个。
  • Class.newInstance()需要构造器必须是可见的，Constructor.newInstance()在一定情况下还可以调用私有类型的构造器。
  • Class.newInstance()通过构造器可以抛出任何类型的异常（checked或者unchecked），Constructor.newInstance(）总是抛出封装了这些异常的InvocationTargetExcetion

在Java中创建对象的五种不同方式,转自5 Different ways to create objects in Java with Examples

  1. 使用new关键字
  2. 调用Class.newInstance()方法
  3. 调用Constructor.newInstance()方法
  4. 调用clone()方法
  5. 使用反序列化
    前三种方法都调用了构造器，第四种和第五种没有调用

Spring In Action Notes

  1. 通过DI，对象的依赖关系将由系统中负责协调各对象的第三方组件在创建对象的时候进行设定。对象无需自行创建或管理它们的依赖关系。
  2. 创建应用组件之间写作的行为通常称为装配（wiring）。
  3. Spring通过应用上下文（Application Context）装载bean的定义，并把他们组装起来。Spring应用上下文全权负责对象的创建和组装。Spring自带了
    多种应用上下文的实现。他们之间主要的区别仅仅在于如何加载配置。
  4. 面向切面编程(Aspect Oriented Programming)。系统由许多不同的组件组成，每个组件各负责一块特定功能。除了实现自身核心的功能之外，这些组
    件还经常承担着额外的职责。注入日志、事务管理和安全这样的系统服务经常融入到自身具有核心业务逻辑的组件中去。这些系统服务通常被称为横切关
    注点，因为他们会跨越系统的多个组件。
    AOP可以使这些服务模块化，并以声明的方式将它们应用到他们需要影响的组件中去。可以使这些组件具有更高的内聚性并且会更加关注自身的业务，
    完全不需要了解涉及系统服务所带来的复杂性。总之AOP能够确保POJO的简单性。
  5. Spring自带了多个容器实现，可以归为两种不同类型。 #Bean工厂# (org.springframework.beans.factory.eanFactory接口定义)和 #应用上下文# (org.
    springframework.context.ApplicationContext接口定义)。Bean工厂是最简单的容器，应用上下文基于BeanFactory构建，并提供应用框架级别的服务。
  6. Spring常用上下文类型：
    AnnotationConfigApplicationContext
    AnnotationConfigWebApplicationContext
    ClassPathXmlApplicationContext
    FileSystemCmlapplicationcontext
    XmlWebApplicationContext
  7. Spring中Bean的生命周期
    实例化 -> 填充属性 -> 调用BeanNameAware的setBeanName()方法()->调用BeanFactoryWare的setBeanFactory方法->调用ApplicationContextAwar的setApplicationContext方法->调用BeanPostProcessor的预初始化方法->调用InitializingBean的afterPropertiesSet方法->调用自定义的初始化方法 -> 调用BeanPostProcessor的初始化后方法 -> bean可以使用了 || 容器关闭 -> 调用DisposableBean的destroy()方法 -> 调用自定义的销毁方法

chapter2

  1. Spring装配机制：
    • 在XML中进行显示配置
    • 在Java中进行显示配置
    • 隐式的bean发现机制和自动装配（最强大）：组件扫描（component scanning）和自动装配(autowiring)
  2. Spring应用上下文中所有的bean都会给定一个id, 默认是将类名的第一个字母变为小写。
    给bean设置id可以用 @Component(“beanName”)或者Java依赖注入规范中的 @Named(“beanName”)注解，有细微差异，大部分可互换，推荐使用@Component

  3. 设置组件扫描的基础包

    • 默认以配置类所在的包为基础包（base package）来扫描组件

    • 指定不同的基础包需要在@ComponentScan的value属性中指明包的名称

    • 多个基础包
